Java Tutorial 4 - De bouwstenen van een Java-programma, klassen en methodes Gepost op 10 aug. 2014

Nu we de basis data types, structuren en ook Strings onder de knie hebben, kunnen we beginnen kijken naar hoe een basis programma in Java eruit ziet.
We gaan overlopen wat een klasse precies is, waarvoor de main-methode wordt gebruikt, alsook gaan we zelf een methode leren schrijven.

Een klasse

Het programmeren in Java is eigenlijk het schrijven van klassen (Classes), waarbij we een klasse definiëren door public naamKlasse{ }.
Maar natuurlijk willen we dat er iets gaat gebeuren als we onze programma -klasse eigenlijk- uitvoeren. Daarom moeten we gebruik maken van de main-methode, deze moet in tenminste één klasse aanwezig zijn. Wanneer je jouw programam wilt uitvoeren zal de Java-interpreter beginnen met te lezen van de main-methode.

Een main-methode ziet er als volgt uit, public static void main(String args[]){ }.
Wat public, static, void doet zal ik later uitleggen wanneer we toe komen aan het schrijven van methodes. Wanneer je jouw Java applicatie start via een Terminal kan u ook argumenten geven aan dit programma, vandaar de code String args[]. Zoals u geleerd hebt in de vorige tutorial betekent die String args[] niets meer dan een array (rij) van Strings met als naam args.

Wanneer je de main-methode schrijft in de klasse heb je een uitvoerbaar programmatje geschreven, dit is weliswaar zeer optimistisch vermits je programma niets doet.

             
public class naamKlasse {
	public static void main(String[] args) {
		//doe iets
	}
}

Methodes

Waar een programmeur veel met te maken gaat hebben is methodes. Methodes zijn acties die een klasse kan uitvoeren. Wanneer je bijvoorbeeld meerdere malen dezelfde -vooral complexe- acties moet uitvoeren, maak je gebruik van methodes. Een voorbeeld ter verduidelijking: 

public class VoorbeeldKlasse {
	public static void main(String[] args) {
		//we willen iets uit printen zonder elke keer opnieuw system.out.println(...); te typen.
        printWatUit();
        printWatUit();
        printWatUit();
	}
	public static void printWatUit() {
		System.out.println("Eerste methode geschreven!");
		
	}
}

In de main-methode wordt de printWatUit()-methode opgeroepen, die dan vervolgens wordt uitgevoerd. We moeten ons niet beperken tot simpele dingen tonen op het scherm maar door die methodes kunnen we complexe berekeningen, op een elegante manier gemakkelijk hergebruiken.

Verduideling van de code

public staat hier voor dat alle klassen gebruik kunnen maken van die methode, static wordt later uitgelegd, wat hier wel zeer belangrijk is is het woord void.
De vertaling van void is leeg/leegte/nietig, de methode gaat in dit geval niets terug geven van informatie, daarvoor staat die void. Als je wilt dat de methode informatie terug geeft zal je void moeten vervangen door het datatype dat de methode teruggeeft.

Een voorbeeld ter verduidelijking: 

public class GemiddeldeBerkenen {
	public static void main(String[] args) {
		//een methode die een double teruggeeft
		double gemiddelde = zoekGemiddelde(10,16); 	//bij deze methode geven we 2 parameters mee
				                                //deze paramaters zijn van het type int
		System.out.println(gemiddelde); // hier gebruiken we geen " " vermits we de waarde van gemiddelde willen tonen
	}
	/*we moeten hier zeggen dat we 2 parameters 
	 * gaan krijgen die beiden een int 
	 * zijn en we geven die parameters 
	 * hier een naam a en b
	 * 
	 * De volgorde is van belang! a slaat hier op die 10 en b op die 16*/
	public static double zoekGemiddelde(int a, int b) { 
		double gem;
		gem = (a+b)/2; //berkenen van het gemiddelde
		return gem;   //geef de waarde van gemiddelde terug
		              //deze wordt dan opgeslagen in gemiddelde in de main-methode
	}
}

Nu is het dus mogelijk om verschillende waarden mee te geven met die methode om het gemiddelde tussen 2 ints te berkenen, zodat we niet telkens opnieuw de berekening moeten neertypen.
